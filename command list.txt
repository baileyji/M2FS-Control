Agent Outline:
An agent handles a hardware subsystem, e.g. the dataloggerAgent handles interfacing
the with three dataloggers in the instrument and the galilAgen handles interface and 
control of the systems attached to a single Galil motion controller.
Agents are run as daemons, started by the instrument boot-up routines,
which are started by udev or init TODO: figure out what stats agents. An agent listens 
for commands via socket set at agent start.

Each agent supports the following commands in addition to any agent-specific commands:
help
quit

help will return a multiline message consisting of available commands and their usage
 

initialize itself
execute a command
help
log
